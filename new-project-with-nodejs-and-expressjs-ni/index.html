Bower

上图可以看出前端如果使用**框架，那必然引入的文件会很多很多，而且如果需要更换版本，哦！！！ 简直是噩梦。所以使用 Bower来管理前端的各种依赖。

安装 node 的时候就已经安装了 bower ，所以现在可以直接使用了，首先在当前项目的根目录下打开 terminal ，输入

bower init

会有一些需要确认的信息，一般回车就好，等到向导结束，就会生成一个 bower.json 文件在项目的根目录如果刚刚 terminal 的路径没有错，生成的文件大概是这样的：

{
  “name”: “jinger”,
  “version”: “0.0.1”,
  “homepage”: “https://github.com/xiaoyee/jinger“,
  “authors”: [
    “Ochukai <sex.xiaoyee@gmail.com>”
  ],
  “description”: “jinger”,
  “main”: “app.js”,
  “moduleType”: [
    “node”
  ],
  “keywords”: [
    “jinger”
  ],
  “license”: “MIT”,
  “ignore”: [
    “*/.“,
    “node_modules”,
    “bower_components”,
    “test”,
    “tests”
  ],
  “dependencies”: {
    “jquery”: “~2.1.3”
  }
}

最重要的节点当然是 dependencies，这里定义了你的项目使用的库及其版本。

使用 bower 安装的依赖默认实在项目根目录下的 bower_components 文件夹，当然是可以修改的了。修改的话，要在项目根目录新建 .bowerrc 文件，新建这个文件需要用 terminal 输入：

$ cd . > .bowerrc

这是官方的一个例子：

{
  “directory”: “app/components/“, // 这里可以规定依赖安装的位置。
  “analytics”: false,
  “timeout”: 120000,
  “registry”: {
    “search”: [
      “http://localhost:8000“,
      “https://bower.herokuapp.com“
    ]
  }
}

更多信息，在http://bower.io/docs/api/
