MySQL 的几个特殊 SQL 语法

发表于 2014-12-28   |   分类于 MySQL

使用了 node 的 mysql 的库，不是 ORM 只是原生的 SQL，名字是 felixge/node-mysql ，这个库不错，挺好用的，比如

var userId  = 1;
var columns = [‘username’, ‘email’];
var sql     = ‘SELECT ?? FROM ?? WHERE id = ?’;

var query = connection.query(sql, [columns, ‘users’, userId], function(err, results) {
// …
});
亦可以这样

var post  = {
    id   : 1,
    title: ‘Hello MySQL’
};

var query = connection.query(‘INSERT INTO posts SET ?’, post, function(err, result) {
// Neat!
});
console.log(query.sql); // INSERT INTO posts SET id = 1, title = ‘Hello MySQL’
上面都是 Readme.md 中的示例代码，但是看到

INSERT INTO posts SET id = 1, title = ‘Hello MySQL’

我发现这种用法我并没有见过，于是去官网看了下(http://dev.mysql.com/doc/refman/5.1/en/insert.html)，确实有这种用法我就放心了。

INSERT INTO tablename SET *

INSERT [LOW_PRIORITY | DELAYED | HIGH_PRIORITY] [IGNORE]
    [INTO] tbl_name
    SET col_name={expr | DEFAULT}, …
    [ ON DUPLICATE KEY UPDATE
      col_name=expr
        [, col_name=expr] … ]

这样的写法有什么好处呢？

  • 传统方法将列名和列值分开了，在使用时，列名必须和列值的数量一致，而且分开后不容易对比
  • 这种方法允许列名和列值成对出现和使用，不会出现顺序和数量不对应的情况
  • 速度快
     

REPLACE INTO tablename SET *

REPLACE [LOW_PRIORITY | DELAYED]
    [INTO] tbl_name
    SET col_name={expr | DEFAULT}, …

在看上面的 INSERT INTO 的时候看到了一句话：

You can use REPLACE instead of INSERT to overwrite old rows.
所以点进去一探究竟，REPLACE 与 INSERT 很相似。
只有一点例外，假如表中的一个旧记录与一个用于PRIMARY KEY或一个UNIQUE索引的新记录具有相同的值，则在新记录被插入之前，旧记录被删除。

注意，除非表有一个PRIMARY KEY或UNIQUE索引，否则，使用一个REPLACE语句没有意义。该 语句会与INSERT相同，因为没有索引被用于确定是否新行复制了其它的行。
